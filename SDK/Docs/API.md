# API Binary SDK

Все функции используют cdecl, не являются потокобезопасными и должны вызываться из одного UI-потока. Числовые ID непрозрачны. DLL не сохраняет указатели на временные параметры после возврата, если явно не указано иное.

Для функций `TV_Copy*` параметр `*Buffer` принадлежит вызывающему, `BufferCharacters` задаёт ёмкость UTF-16 в символах, а результат включает завершающий ноль. Остальные функции не передают вызывающему память, требующую освобождения.

## Версия

### TV_CheckAPIVersion

- Сигнатура: TV_CheckAPIVersion.i(ExpectedVersion.i)
- Назначение: Выполняет публичную операцию бинарного SDK.
- Параметры: ExpectedVersion.i; значения проверяются DLL.
- Возвращает: Целое значение, `#False`/0 либо ID согласно назначению.
- Ошибки: при отказе используйте `TV_GetLastError()` и `TV_CopyLastErrorText()`; неверные ID и параметры отвергаются.
- Владение памятью: память вызывающего остаётся у вызывающего; освобождать память DLL не требуется.
- До TV_Init(): Да
- После TV_Shutdown(): Да
- Потокобезопасность: нет, вызывать из единственного UI-потока.
- Пример: Result = TV_CheckAPIVersion(...); используйте реальные параметры из примеров SDK.
