# Руководство backend

[English version](en/BackendGuide.md)

В бинарном SDK реализован только `#TV_BACKEND_WINDOWS_CONSOLE`. Выберите его
через `TV_SelectBackend()` до `TV_Init()`.

Режимы консоли:

- `CURRENT` использует текущую или родительскую консоль;
- `NEW` создаёт отдельную консоль;
- `AUTO` предпочитает текущую и использует новую как fallback.

Backend восстанавливает режимы, viewport, содержимое, заголовок и курсор при
`TV_Shutdown()`. Нельзя перенаправлять TUI-вывод в pipe или файл.

Если console input объединяет движение и изменение кнопки мыши, SDK сохраняет
нажатие/отпускание как приоритетное событие. Переход кнопки не превращается в
последующий фантомный клик по нижнему окну.

Оконные тени состоят из полноразмерных клеток и не требуют font detection.
Символ затеняемой ячейки сохраняется и в Classic theme рисуется тёмно-серым на
чёрном, поэтому текст заднего окна остаётся видимым как приглушённый.
Backend проверяет glyph coverage только для resize handle. CP866/1251 не
является критерием, поскольку кадр выводится через `WriteConsoleOutputW()`.

Маркер resize `◢` также проверяется отдельно. Если glyph отсутствует, SDK
показывает OEM-совместимый `▓`, не меняя hit-testing правого нижнего угла.

Window shadow занимает две полные клетки справа и одну полную строку снизу.
Raised-button shadow использует стандартные `▄` и `▀`.

Регистрация пользовательских backend не входит в закрытый ABI, поскольку она
потребовала бы раскрытия внутренних буферных структур. Serial/ESP и COM-порты
имеют статус «в разработке».
