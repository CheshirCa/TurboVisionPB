# Модель событий

[English version](en/EventModel.md)

`TV_PollEvent()` неблокирующе получает одно событие, `TV_WaitEvent()` допускает
таймаут. Перед анализом прикладной код вызывает `TV_DispatchEvent()` — он
маршрутизирует ввод окнам и controls и может преобразовать событие в
`TV_EVENT_COMMAND`.

`Command` содержит код операции, `Source` — числовой ID control или окна.
Alt+F3, Alt+F5 и Ctrl+F6 используют ту же командную модель. При модальном окне
ввод нижних окон блокируется. `TV_Event` принадлежит вызывающему; DLL не
сохраняет переданный указатель после возврата функции.

Для экранного `TV_EVENT_RESIZE` поле `Source` равно нулю, `MouseX` содержит
новую ширину, а `MouseY` — новую высоту. К моменту возврата события
`TV_GetScreenSize()` уже сообщает новую геометрию; оба виртуальных буфера
пересозданы и следующий render выполняет полный кадр. Ненулевой `Source` у
resize означает WindowID отдельного окна.

Desktop menu и системное меню окна взаимоисключающие. При переходе между ними
предыдущий popup закрывается; это правило сохраняется после screen resize и
оконных Cascade/Tile.

Windows Console может передать перемещение и изменение кнопки одним input
record. В таком случае SDK возвращает `MOUSE_DOWN` или `MOUSE_UP`; чистый
`MOUSE_MOVE` формируется только без перехода кнопки.

Глобальные shortcuts menu/status доступны при открытом popup другого меню.
Совпадение закрывает popup и формирует обычный `TV_EVENT_COMMAND`, поэтому
Alt+X в classic demo всегда достигает обработчика Exit.

Библиотека однопоточная. Не вызывайте событийный цикл параллельно и не
передавайте один `TV_Event` нескольким потокам.
