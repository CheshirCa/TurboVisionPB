# Журнал изменений TurboVisionPB

## 0.16.8 — 2026-07-16

- закрытие последнего окна и изменение desktop popup инвалидируют виртуальный
  desktop, поэтому старые окно и меню не остаются в кадре;
- глобальные menu/status shortcuts обрабатываются даже при открытом другом
  desktop menu; Alt+X закрывает popup и формирует Exit command;
- `TemplateWindowsTests` закрывает последнее обычное окно через системное меню,
  проверяет очистку окна/popup и Alt+X при открытом Window menu на x64/x86;
- публичный API и ABI не изменились; ESP32, Serial backend и COM-порты не
  затрагивались.

## 0.16.7 — 2026-07-16

- Calendar mouse handler теперь проверяет верхнее окно в координате события до
  обработки сетки дней;
- клик по перекрывающему Calendar заголовку Calculator больше не проходит
  сквозь Calculator, не меняет выбранную дату и не активирует Calendar;
- `TemplateWindowsTests` воспроизводит точную геометрию demo: Calendar `(1,2)`,
  Calculator `(8,6)` и конфликтующий клик `(15,6)` на x64/x86;
- публичный API и ABI не изменились; ESP32, Serial backend и COM-порты не
  затрагивались.

## 0.16.6 — 2026-07-16

- Windows Console mouse records теперь сначала проверяются на изменение
  состояния кнопок, даже если одновременно установлен флаг движения;
- устранены потерянные mouse down/up и возможный отложенный переход кнопки на
  следующей координате;
- `TemplateWindowsTests` проверяет комбинированные move+down/move+up records,
  верхнее окно Calculator и безопасное наведение после закрытия меню на x64/x86;
- публичный API и ABI не изменились; ESP32, Serial backend и COM-порты не
  затрагивались.

## 0.16.5 — 2026-07-16

- desktop menu и системное меню окна сделаны строго взаимоисключающими;
- устранено наложение popup после resize и переключения между menu;
- Close Calculator больше не поглощается оставшимся открытым desktop menu;
- `TemplateWindowsTests` воспроизводит resize, закрытие Calendar, переключение
  popup и закрытие Calculator с полной очисткой menu state на x64/x86;
- публичный API и ABI не изменились; ESP32, Serial backend и COM-порты не
  затрагивались.

## 0.16.4 — 2026-07-16

- обычные окна после сильного console resize ограничиваются рабочей областью
  между menu bar и status bar, поэтому верхняя рамка не скрывается;
- classic desktop source/public examples закрывают Editor по `TV_CMD_CLOSE`
  из системного меню, в том числе после resize и Window/Tile;
- системные заголовки запускаемых примеров показывают текущую версию;
- release EXE получают штатный PureBasic `/USER` manifest, поэтому Windows не
  требует elevation для x86 `Example05_InstallerWrapper.exe`;
- `WindowManagementTests` проверяет resize round trip, Tile и системный Close
  обычного окна на x64/x86;
- публичный API и ABI не изменились; ESP32, Serial backend и COM-порты не
  затрагивались.

## 0.16.3 — 2026-07-16

  desktop, а не физической строкой `Y = 0`;

  временно клипуется без изменения размера;
- TemplateWindows regression test проверяет уменьшение до 40×15 и обратное
  восстановление Y/размера окна на x64/x86;
- публичный API и ABI не изменились; ESP32, Serial backend и COM-порты не
  затрагивались.

## 0.13.8 — 2026-07-16

- оконная тень возвращена к геометрии оригинального Turbo Vision: две полные
  символьные клетки справа и одна полная строка снизу со смещением 2×1;
- raised-кнопки сохраняют компактную полуклеточную тень `U+2584` справа и
  `U+2580` снизу;
- удалены quadrant/quarter/OEM capability branches оконной тени; font coverage
  проверяется только для resize handle;
- тесты проверяют точную полноразмерную геометрию окна и полутень кнопки;
  публичный ABI не изменился.

## 0.13.7 — 2026-07-16

- правая тень однострочной raised-кнопки начинается с нижнего half-block, а не
  с вертикального full-height half-block;
- точный Unicode-вариант использует `U+2598` — верхнюю левую четверть — во
  внешнем нижнем правом углу окна и кнопки;
- поскольку CP866 не содержит позиционной четверти, Raster/OEM fallback
  использует light shade B0 (`U+2591`) как четверть плотности без разрыва;
- glyph detection и regression tests обновлены; публичный ABI не изменился.

## 0.13.6 — 2026-07-16

- OEM shadow fallback больше не рисует выступающую верхнюю заглушку `U+2584`
  и полноразмерный угловой хвост `U+2588`;
- правая и нижняя полосы используют только CP866 DD/DF, представленные в
  Unicode-буфере как `U+258C` и `U+2580`, и смещены на строку/колонку;
- внешняя угловая клетка остаётся нетронутой; raised-кнопка сохраняет правую
  полуполосу при высоте в одну строку;
- regression tests проверяют пустые endpoint/corner cells; ABI не изменился.

## 0.13.5 — 2026-07-16

- Raster/Terminal console font теперь всегда использует OEM shadow fallback,
  что исключает ложноположительный GDI font substitution для quadrant glyphs;
- поддержка resize handle `U+25E2` проверяется отдельно от shadow glyphs;
- при отсутствии `U+25E2` нижний правый угол resizable-window показывает
  CP437-совместимый `U+2593` вместо boxed question;
- расширен `WindowManagementTests`; публичный ABI не изменился.

## 0.13.4 — 2026-07-16

- восстановлена точная полуклеточная тень 0.13.2 для console fonts, содержащих
  Unicode quadrant glyphs;
- Windows backend определяет активный console font через
  `GetCurrentConsoleFontEx()` и проверяет glyph coverage через
  `GetGlyphIndicesW()`;
- для ограниченных Raster/OEM fonts используется связный fallback из
  `U+2584`, `U+258C`, `U+2580` и `U+2588` без boxed-question glyphs;
- output code page не используется как критерий: кадр выводится Unicode API
  `WriteConsoleOutputW()`; public ABI не изменился.
